# Embedding Cache Pre-filling Script

## 1. 개요 및 목적
이 디렉토리는 오픈 전 **DB 임베딩 캐시(`search_query_cache`)를 사전에 채워 넣기 위한 스크립트와 문서**를 보관합니다.

*   **목적:** 사용자가 태그 검색창에 특정 단어를 처음 검색할 때 발생하는 구글 Gemini API 호출 지연(약 300ms)을 예방하기 위해, 예상 가능한 모든 자전거 관련 검색어의 임베딩(벡터) 값을 DB에 미리 생성해 둡니다.
*   **결과:** 오픈 당일 수많은 유저가 접속하여 다양한 단어를 검색하더라도, 99% 이상의 확률로 Cache Hit가 발생하여 0~1ms 만에 즉시 검색 결과를 반환하게 됩니다.

---

## 2. 데이터 수집 파이프라인 (Pipeline)

### 2.1 시드 데이터 수집 (올드 패션 방식)
정확하고 이미 시스템에 존재하는 데이터를 기반으로 기본 검색어 풀을 구성합니다.
1.  **기존 태그 (`tags` 테이블):** 약 236개의 고유 슬러그 (예: 한강, 벚꽃, 힐클라임)
2.  **지역명/고유명사 (`waypoints` 테이블):** 전체 웨이포인트 이름을 형태소 단위로 분리한 후, 파이썬 `Counter`를 이용해 최빈출 단어 상위 300개를 추출합니다.

### 2.2 AI 기반 창의적 브레인스토밍 (Gemini 3.1 Pro + Google Search)
기존 DB 데이터가 커버하지 못하는 최신 트렌드, 감성어, 은어를 수집하기 위해 구글 검색 툴을 장착한 AI 모델을 활용합니다.

*   **모델:** `gemini-3.1-pro-preview` (AI Studio API 키 사용)
*   **도구:** `google_search`
*   **실행 방식 (Multi-threading):** 
    *   10개의 스레드(Worker)를 동시에 띄워 병렬로 AI에게 프롬프트를 전송합니다.
    *   이 10개의 스레드 배치를 **총 20회 반복(Loop)** 실행합니다.
    *   즉, 총 200번의 AI 호출이 이루어지며, 1회당 30개씩 추출하므로 이론상 **최대 6,000개**의 무작위 단어 풀(Pool)을 확보하게 됩니다. (중복 제거 전)
*   **프롬프트 특징:** 모델이 스스로 구글 검색어를 결정하여 자유롭게 최신 커뮤니티 글이나 유튜브 제목을 탐색하도록 유도합니다.
*   **로깅:** 모델이 실제로 사용한 구글 검색어(`used_search_queries`)를 JSON 응답에 포함하게 하여, 파이썬 콘솔에서 모델의 검색 다양성을 실시간으로 모니터링할 수 있도록 합니다.

### 2.3 데이터 정제 (Deduplication)
수집된 약 800개의 단어 리스트를 파이썬 스크립트 단에서 정제합니다.
*   1글자 단어 제거
*   `set()`을 이용한 중복 단어 제거
*   이미 DB(`search_query_cache`)에 존재하는 단어 필터링 (API 비용 절감)

---

## 3. 병렬 처리 및 캐싱 (Multi-threading)
최종 정제된 약 400~500개의 알짜 키워드를 DB에 초고속으로 캐싱합니다.

*   **방식:** `concurrent.futures.ThreadPoolExecutor`를 사용하여 10개의 워커(Worker) 스레드를 동시 실행.
*   **모델:** `gemini-embedding-001` (백엔드와 동일하게 Vertex AI 환경 사용).
*   **저장:** 변환된 3072차원의 임베딩 배열을 `ON CONFLICT DO NOTHING` 조건과 함께 DB에 `INSERT`.
*   **진행률 표시:** `tqdm`을 활용하여 콘솔에 실시간 진행 상황 및 성공/실패 통계 출력.

## 4. 프롬프트 명세 (Prompt Specification)

AI 브레인스토밍 단계에서 사용되는 시스템 프롬프트는 다음과 같습니다.

```text
너는 한국의 자전거 라이딩 코스를 추천하는 전문가야. 지금 네게는 구글 검색 툴이 주어져 있어.

[지시 사항]
1. 구글 검색 툴을 사용해서 한국 자전거 동호인들의 최신 커뮤니티 글, 블로그 후기, 유튜브 영상 제목 등을 자유롭게 검색해서 읽어봐. 매 호출마다 완전히 새롭고 다양한 주제(예를 들어, 어느 날은 식도락 투어, 어느 날은 극강의 산악 훈련 등)를 스스로 설정해서 검색해.
2. 검색한 문서를 바탕으로, 한국의 자전거 라이더들이 코스나 장소를 찾을 때 머릿속에 떠올릴 만한 연관 키워드 30개를 추출해 줘.
3. 추출 대상: 지명, 코스 고유명사, 지형/난이도, 라이딩 목적/스타일, 동호인 은어, 먹거리, 경치 등 '자전거 라이딩'과 관련된 1~3어절의 모든 자유로운 단어.

[출력 규칙]
반드시 아래의 JSON 형식으로만 응답해. 네가 실제로 정보를 찾기 위해 사용했던 구글 검색어(1개 이상)를 `used_search_queries` 배열에 반드시 포함시켜줘.

{
  "used_search_queries": ["실제로 네가 구글링에 사용한 검색어1", "검색어2"],
  "keywords": ["단어1", "단어2", "단어3", ...]
}
```

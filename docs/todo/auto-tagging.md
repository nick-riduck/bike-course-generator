# 자동 태그 생성 기능 설계

## 개요
코스 저장 시 Gemini 2.5 Flash Lite를 활용하여 자동 태그를 제안하는 기능.

## 핵심 설계 원칙
- **LLM 호출은 저장 시 1회만** (비용/성능 최적화)
- **프롬프트는 비동기로 사전 구성** (경로 변경 시마다 갱신)
- **자동 태그는 "제안"** (기본 선택 상태, 클릭으로 해제 가능)
- 사용자 수동 태그와 공존

## 아키텍처

### 데이터 흐름
```
[프론트엔드]
경로 편집 중 (시작점/경유지 변경)
  → debounce 후 프롬프트 갱신 (LLM 호출 X, 로컬 state만)
  → 프롬프트 = { 좌표 샘플, distance, ascent, 고도 프로필 요약 }

저장 모달 열림
  → 기존 수동 태그 표시
  → "AI 태그 제안" 버튼 or 자동 호출
  → Backend API 호출 (사전 구성된 프롬프트 전달)
  → Gemini Flash Lite 1회 호출 (~500ms 목표)
  → 제안 태그 표시 (기본 선택 상태, 토글 가능)
  → 사용자 확인 후 저장

[백엔드 - Cloud Run]
POST /api/routes/suggest-tags
  → 프롬프트 조립 + Gemini 2.5 Flash Lite 호출
  → 태그 리스트 반환
```

### 프롬프트 구성 (프론트에서 사전 준비)
경로 변경 시마다 아래 데이터를 갱신해둠:
- 시작점/끝점 좌표 (위경도)
- 경유 좌표 샘플 (10-20개로 다운샘플링)
- 총 거리 (km)
- 총 오르막 (m)
- 고도 프로필 요약 (최저/최고/평균 고도)
- (선택) 주요 구간별 경사도

### 저장 모달 UI 변경
```
┌─────────────────────────────────────┐
│ Tags                                │
│ ┌─────────────────────────────────┐ │
│ │ #한강 #장거리  [태그 입력...]    │ │  ← 수동 태그 (기존)
│ └─────────────────────────────────┘ │
│                                     │
│ AI Suggested Tags        [🔄 새로고침]│
│ ┌─────────────────────────────────┐ │
│ │ ✅ #서울  ✅ #한강변  ✅ #평지   │ │  ← 제안 태그 (토글)
│ │ ☐ #출퇴근  ✅ #초보추천          │ │
│ └─────────────────────────────────┘ │
└─────────────────────────────────────┘
```

## 구현 단계

### Phase 0: Gemini Flash Lite 성능 테스트
- [ ] Gemini 2.5 Flash Lite API 연동 테스트
- [ ] 응답 시간 측정 (목표: < 1초)
- [ ] 프롬프트 최적화 (한국 자전거 코스 도메인)
- [ ] 태그 품질 검증

### Phase 1: 백엔드 API
- [ ] `POST /api/routes/suggest-tags` 엔드포인트 구현
- [ ] Gemini Flash Lite 클라이언트 설정 (Cloud Run 환경)
- [ ] 프롬프트 템플릿 설계
- [ ] 응답 파싱 + 기존 태그 DB와 매칭 (유사 태그 통합)

### Phase 2: 프론트엔드 프롬프트 사전 구성
- [ ] 경로 변경 시 태그 프롬프트 데이터 갱신 로직
- [ ] debounce 처리
- [ ] 프롬프트 데이터 state 관리

### Phase 3: 저장 모달 UI
- [ ] AI 제안 태그 영역 추가
- [ ] 제안 태그 토글 (기본 선택, 클릭 해제)
- [ ] 수동 태그와 제안 태그 merge 로직
- [ ] 로딩 상태 UI (skeleton/spinner)
- [ ] 새로고침 버튼

### Phase 4: 통합 및 최적화
- [ ] 에러 핸들링 (LLM 실패 시 graceful fallback)
- [ ] 기존 태그 embedding과 LLM 제안 태그 매칭
- [ ] 캐싱 (동일 경로 재저장 시 재호출 방지)

## 기술 선택
- **LLM**: Gemini 2.5 Flash Lite (Cloud Run 환경에 적합, 저비용, 빠름)
- **호출 방식**: 백엔드에서 Gemini API 직접 호출
- **프롬프트 전략**: 좌표 + stats 기반, 한국 자전거 도메인 특화

## 미결정 사항
- 제안 태그 개수 제한 (5개? 10개?)
- 기존 DB 태그와 유사도 매칭 threshold
- 새 태그 자동 생성 허용 vs 기존 태그만 제안
